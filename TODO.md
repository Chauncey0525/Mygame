## 主策划 · 交互设计需求（设计组/前端协作）

> 以下由主策划提出，供设计组输出交互规范/原型/说明，前端或程序实现。包含反馈、动效、流程、情感化与无障碍等。

### 反馈与提示
- [ ] **Toast / 轻提示**：任务领取成功、资源到账、设置保存等用页面内 Toast（右上或居中，几秒自动消失），替代 `alert`。设计：出现/消失动画、样式与中世纪主题一致。
- [ ] **加载状态**：所有发起请求的按钮（召唤、领取、战斗结算等）在请求期间显示 loading（旋转图标或按钮内文案变为「加载中」），避免重复点击。
- [ ] **错误与空状态**：网络错误、接口失败时统一提示样式与文案；列表无数据时展示「空状态」插画+引导文案（如「还没有角色，去召唤吧」），而非空白。
- [ ] **二次确认**：消耗钻石/体力、退出登录等关键操作，弹确认框（文案清晰、取消/确定主次分明），与设置中「确认弹窗」开关联动。

### 点击、悬停与动效
- [ ] **按钮态**：所有可点击按钮具备 hover（高亮/轻微上浮）、active（按下缩小或变暗）、disabled（灰化+不可点）的视觉与交互规范。
- [ ] **卡片/建筑入口**：城镇建筑、角色卡、关卡卡等 hover 时有明显反馈（边框/阴影/轻微放大），点击有按下感（scale 或闪一下）。
- [ ] **弹窗动效**：弹窗打开为缩放+淡入，关闭为淡出；遮罩可点击关闭时，点击遮罩有关闭反馈。时长与「减少动画」设置联动。
- [ ] **资源变化反馈**：获得金币/钻石/体力时，资源栏有「+N」飘字或短时高亮/数字滚动，强化获得感。

### 列表、滚动与加载
- [ ] **长列表**：图鉴、角色列表等数量多时，考虑虚拟滚动或分页，避免一次渲染过多 DOM。设计滚动条样式与主题一致。
- [ ] **骨架屏 / 占位**：进入角色页、图鉴、副本列表时，数据未返回前展示骨架屏或占位块，避免白屏或布局跳动。
- [ ] **下拉刷新 / 加载更多**：若列表支持刷新或分页，明确交互方式（如底部「加载更多」按钮或滚动到底加载）。

### 表单与输入
- [ ] **输入框态**：获得焦点时边框/外发光与主题色一致；错误时红色边框+下方错误文案；必填项未填时提交有明确提示。
- [ ] **登录/注册**：记住我、忘记密码、验证码倒计时等交互清晰；错误提示靠近表单项，不只用 alert。
- [ ] **数值/选项**：滑块、开关、下拉框的交互态（拖拽、选中）与中世纪 UI 风格统一，且支持键盘操作。

### 导航与流程
- [ ] **面包屑 / 层级**：多级页面（如 城镇 → 军营 → 角色详情）有清晰层级，返回按钮或面包屑可回到上一级，当前页在导航上高亮。
- [ ] **多步流程**：召唤十连、战斗结算、任务领取等多步流程，每步有进度感（步骤条或动画顺序），支持跳过时明确「跳过」入口。
- [ ] **ESC 与焦点**：弹窗打开时焦点进入弹窗、ESC 关闭弹窗、焦点不跑到遮罩后元素（焦点陷阱），与现有设置弹窗行为一致并推广到所有弹窗。

### 情感化与引导
- [ ] **首次进入**：新玩家首次进城镇或关键功能时，可做简短引导（高亮建筑+一句文案），可跳过且仅展示一次（本地记录）。
- [ ] **空状态**：无角色、无任务、无关卡时，用插画+一句引导文案+主操作按钮，而非纯文字。
- [ ] **成就时刻**：升级、首次通关、获得传说角色等，有简短庆祝动效（如全屏闪光+标题+音效），可跳过但默认播放一次。
- [ ] **加载/过渡**：进入战斗、切换场景时的短过渡（黑屏淡入淡出或简单 logo 动画），避免硬切。

### 无障碍与多端
- [ ] **键盘**：主要流程支持键盘完成（Tab 切换、Enter 确认、ESC 关闭）。焦点顺序合理，焦点环样式与主题一致。
- [ ] **读屏**：关键按钮、图标、弹窗具备 `aria-label` 或 `role`，图片有 `alt`，动态内容变更时可选 `aria-live` 提示。
- [ ] **减少动画**：设置中「减少动画」开启时，所有非必要动效缩短或关闭，保留仅与功能相关的反馈（如受击闪一下）。
- [ ] **触屏**：关键点击区域不小于 44×44；长列表支持触摸滑动；必要时提供「点击两次确认」防误触。

设计组可产出交互说明文档或原型（含状态、动效时长、文案），前端/程序按条实现并在本区块勾选。

## 使用说明

1. **勾选**：完成某项后把该行的 `- [ ]` 改为 `- [x]`，可把该条移到「已完成」区。
2. **进行中**：同一时间建议只列 1～3 项，完成后清空或从待办补充。
3. **策划需求**：详细描述见 [策划需求清单](docs/策划需求清单.md)，此处仅作任务引用与勾选。
4. **设计组**：「主策划 · 设计组美术资源需求」由主策划维护，设计组按条产出 AI 美术资源并勾选，资源放入约定目录后可在 README 注明。
5. **交互设计**：「主策划 · 交互设计需求」供设计组输出规范/原型、前端实现，涵盖反馈、动效、流程、情感化与无障碍等。
6. **持续需求池**：「主策划 · 持续需求池」用于一直提需求、只增不减；排期时把某条勾选并挪到「待办」或「进行中」即可。
7. **同步事项**：「后续更新需同步进行的事项」为每次迭代/发版时的**核对清单**，按触发时机检查文档、数据、资源、版本是否同步，非一次性勾选任务。
8. **新增**：新想法可写在对应分类下，或先记在策划需求清单再在此处加一条待办。
