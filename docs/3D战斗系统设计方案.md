# 3D战斗系统设计方案

## 一、技术选型

### 1.1 核心技术栈
- **Three.js**: WebGL 3D渲染引擎
- **粒子系统**: 自定义粒子特效
- **动画系统**: GSAP + Three.js Animation

### 1.2 模型方案
由于制作专业3D模型成本较高，采用**低多边形(Low-Poly)风格**：
- 风格统一、成本低
- 性能友好
- 适合网页游戏

### 1.3 特效方案
- 粒子特效：火焰、冰霜、雷电等
- 攻击动画：冲锋、挥砍、施法
- 技能特效：根据元素类型定制

## 二、系统架构

```
static/js/
├── battle3d/
│   ├── core/
│   │   ├── scene.js        # 场景管理
│   │   ├── camera.js       # 相机控制
│   │   └── lighting.js     # 光照系统
│   ├── characters/
│   │   ├── modelFactory.js # 模型工厂
│   │   ├── character.js    # 角色基类
│   │   └── animations.js   # 角色动画
│   ├── effects/
│   │   ├── particleSystem.js # 粒子系统
│   │   ├── skillEffects.js   # 技能特效
│   │   └── hitEffects.js     # 受击特效
│   └── battle3d.js         # 主入口
```

## 三、角色模型设计

### 3.1 职业模型分类

| 职业 | 模型特征 | 武器 |
|------|----------|------|
| 战士 | 厚重装甲、盾牌 | 剑/斧 |
| 法师 | 长袍、法杖 | 法杖 |
| 弓箭手 | 轻甲、披风 | 弓箭 |
| 刺客 | 紧身衣、双刀 | 匕首 |
| 骑兵 | 骑马、长枪 | 长枪 |
| 治疗者 | 圣袍、光环 | 法球 |

### 3.2 模型层级结构
```
Character (Group)
├── Body (Mesh) - 身体
├── Head (Mesh) - 头部
├── Weapon (Mesh) - 武器
├── Element Aura (Particle) - 元素光环
└── Shadow (Mesh) - 阴影
```

## 四、技能特效设计

### 4.1 元素特效类型

| 元素 | 特效描述 | 颜色 |
|------|----------|------|
| 火 | 火焰喷射、爆炸 | #ff6b35, #ff8c42 |
| 水 | 水波、冰晶 | #4fc3f7, #81d4fa |
| 风 | 龙卷、风刃 | #81c784, #a5d6a7 |
| 土 | 岩石崩裂、地震 | #a1887f, #bcaaa4 |
| 光 | 圣光、光束 | #fff59d, #ffee58 |
| 暗 | 暗影、黑洞 | #7e57c2, #9575cd |

### 4.2 技能动画流程
1. **准备阶段** (0.3s): 角色蓄力动画 + 元素聚集
2. **释放阶段** (0.2s): 特效飞向目标
3. **命中阶段** (0.3s): 爆炸/受击特效 + 伤害数字
4. **恢复阶段** (0.2s): 角色归位

- 粒子：环境粒子（尘埃、光点）

## 六、性能优化

### 6.1 模型优化
- LOD (Level of Detail): 根据距离切换精度
- 实例化渲染: 相同模型共享几何体
- 骨骼缓存: 动画骨骼复用

### 6.2 特效优化
- 对象池: 粒子系统复用
- 批量渲染: 合并相同材质
- 按需加载: 战斗结束后释放资源

### Phase 2 - 特效系统
1. 六元素技能特效

3. 伤害数字动画

### Phase 3 - 完善
1. 更多角色模型
2. 高级技能特效
3. 场景背景切换
